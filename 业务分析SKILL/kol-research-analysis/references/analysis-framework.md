# KOL Research Analysis Framework

## 1. 信息来源

| 来源 | 读取目标 | 公开仓库处理 |
| --- | --- | --- |
| 飞书目录和周会文档 | 找到项目背景、历史决策、问题反复出现的位置 | 只记录目录类型和方法，不记录原文和链接 |
| KOL 投放表 | 读取成本、例子、约课、到课、转化、ROI | 只保留字段框架，不提交真实数值 |
| 视频和图片素材 | 判断脚本、场景、开头钩子、信任背书 | 可写抽象结构，不放截图和原素材 |
| BI/销售后链路 | 判断线索质量和承接问题 | 只输出脱敏结论，不放导出表 |

## 2. 评估维度

| 维度 | 关键问题 |
| --- | --- |
| 素材资产 | 是纯口播、家庭场景、孩子学习场景、访谈、直播切片，还是图片素材？ |
| 授权范围 | 是否能二次剪辑、dark post、跨平台投放、延长授权期？ |
| 前端获客 | CPA 是否稳定，是否只在短期或单账户有效？ |
| 约课质量 | 约课率是否被区域、人群、表单、落地页或销售承接影响？ |
| 到课转化 | 到课率、到转率和 ASP 是否支持继续放量？ |
| 复投价值 | 素材是否可复剪，是否有新脚本空间，是否已出现疲劳？ |
| 区域差异 | 港澳、台湾、新加坡等市场是否呈现不同前端/后端排序？ |

## 3. 结论分层

| 层级 | 定义 | 动作 |
| --- | --- | --- |
| A 类强资产 | 前端和后链路都稳定，可复剪、可续约、可作为基础盘 | 争取长期授权，做多版本剪辑和落地页联动 |
| B 类测试资产 | 某一维度表现好，但成本、约课或转化存在短板 | 小预算验证，明确止损线 |
| C 类品牌资产 | 有知名度或信任价值，但直接转化不稳定 | 用于品牌背书、落地页信任和再营销 |
| D 类暂停资产 | 前端、后链路或授权条件无法支撑复用 | 停止放量，保留历史经验 |

## 4. 输出格式

每次复盘建议输出：

- 项目背景：本轮评估的区域、时间段、渠道范围。
- 关键发现：3-5 条可执行判断。
- KOL 分层：A/B/C/D 类及理由。
- 素材动作：复剪、重拍、换脚本、换落地页或暂停。
- 后链路动作：约课、到课、转化、销售承接的具体优化点。
- 风险说明：数据权限、口径差异、样本不足、授权期限。

## 5. 公开安全写法

可以写：

- “某中腰部亲子类 KOL 在早期素材中约课质量较好，但后续出现疲劳。”
- “图片素材更依赖落地页信任承接，视频素材更适合作为主投素材验证。”
- “不同区域的前端成本和后链路排序不同，需要分区评估。”

不要写：

- 真实 KOL 姓名、报价、合同条款、授权价格。
- 真实成本、GMV、ROI、投放账户、飞书链接。
- 内部老师、客户、学生、家长或员工姓名清单。
