# Reviewer Notes

These notes are for teammates who evaluate repository progress from the outside.

## What To Count As Progress

Count these as meaningful progress:

- New reusable skill instructions.
- New scripts with example input configuration.
- Better README, security, or quality documentation.
- Clearer business use case and deliverable standard.
- Safer handling of public/private boundaries.
- Tests or fake sample outputs that help others verify the workflow.

Do not overvalue:

- Commit count alone.
- Large file count without explanation.
- Raw exports, logs, screenshots, or local cache files.
- Private operational data uploaded to public GitHub.

## Business Value Lens

The strongest business value in this repository is repeatable public feedback collection.

Useful questions:

- Can a teammate understand what business problem this solves?
- Can a teammate reuse the skill or script without reading private chat history?
- Is the output shape useful for market, product, operation, or management decisions?
- Are platform limits and safety boundaries clear?
